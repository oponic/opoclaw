# Plugins

This page documents the opoclaw plugin system: manifest format, lifecycle API, examples, security guidance, and CI checks.

## Overview

Plugins let you extend opoclaw by adding OpenAI/OpenRouter-compatible `function` tools. Plugins live in the workspace `plugins/` directory and are discovered at startup when `enable_plugins` is set in `config.toml`.

- Manifest: `plugin.json` placed in `workspace/plugins/<plugin>/plugin.json`.
- Entry: `entry` points to the plugin entry file (TypeScript/JavaScript) — typically `plugin.ts`.
- Runtime contract: export `tools` and `invoke(name, args, context)`.

## Enabling plugins

1. Run the onboarding wizard (it creates `workspace/plugins/` and a sample plugin).
2. In `workspace/config.toml` set:

```
enable_plugins = true
plugin_dir = "workspace/plugins"
```

Worker execution is now the only supported runtime for plugins.

## Manifest (plugin.json)

Minimal example:

```json
{
  "name": "example-echo-plugin",
  "version": "0.1.0",
  "entry": "plugin.ts",
  "description": "Example plugin: adds one tool."
}
```

Key fields

- `name` (string, required): unique plugin id.
- `version` (string): semver of the plugin.
- `entry` (string): entry file relative to plugin directory (defaults to `plugin.ts`).

Legacy fields are rejected: `permissions`, `mounts`, and `hooks`.

## Plugin API

Plugins must export:

- `tools`: array of OpenAI `function` tool descriptors.
- `invoke(name, args, context)`: handler called by runtime when a plugin tool is invoked.
- optional `deactivate()` for cleanup.

No direct runtime context API is provided (`registerTool`, `readFile`, `editFile`, `checkPermission`, and hooks are removed).

Type helpers are exported from `src/plugin_api.ts` (`OpenAIFunctionTool`, `PluginModule`, and related protocol/message types).

## Example plugin (plugin.ts)

The onboarding scaffold includes a minimal plugin at `workspace/plugins/example-echo-plugin`.
There is also an authoring template at `workspace/plugins/_template`.

Example `plugin.ts`:

```ts
export const tools = [
  {
    type: "function",
    function: {
      name: "echo",
      description: "Echo text",
      parameters: {
        type: "object",
        properties: { text: { type: "string" } },
        required: ["text"]
      }
    }
  }
];

export async function invoke(name, args) {
  if (name === "echo") {
    return `echo:${String(args?.text || "")}`;
  }
  throw new Error(`Unknown tool: ${name}`);
}
```

For strict typing, the template uses `satisfies PluginModule` from `src/plugin_api.ts`.

## Worker-based execution

Each plugin runs in its own isolated Worker process:

- **Isolation**: Plugins run in a separate Bun worker, preventing crashes from affecting the main opoclaw process.
- **Communication**: Host and worker communicate via a minimal protocol: `ready`, `invoke`, `invokeResult`, `shutdown`.
- **Activation**: Worker loads plugin entry file, validates exported `tools`, and exposes `invoke`.
- **Shutdown**: When a plugin is unloaded or opoclaw shuts down, the worker is terminated.

Errors in worker do not crash the main process.

## Security & Permissions

Plugins can only contribute tools. Workspace file and host mutation APIs are not exposed to plugins.

## CI & Tests

Add the following checks to CI:

- Manifest validation: `tests/plugins.test.ts` should validate manifest schema and ensure no path traversal in `entry`.
- Unit tests: plugin discovery and `registerTool` behavior.
- Security tests: ensure legacy APIs are rejected and only valid `function` tools are accepted.
- Lint & Typecheck: `tsc` and ESLint.

## Best practices for plugin authors

- Keep `invoke()` deterministic and cheap.
- Return plain strings or JSON-serializable values from handlers.
- Validate arguments defensively inside plugin code.
- Provide a short `README.md` inside the plugin folder describing usage.

## Troubleshooting

- If a plugin fails to load, check opoclaw logs — imports may fail if TypeScript isn't compiled. Prefer shipping compiled JS for production plugins.
- Use `enable_plugins = false` to disable plugins quickly.
