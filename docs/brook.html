Complying with latest Brook regulations on the use of LLM bots

Editing your configuration

Let's take a look at a typical config.toml.

enable_reasoning = true
reasoning_summary = true
reasoning_summary_model = "anthropic/claude-3-haiku"
ollama_semantic_search = true
authorized_user_id = "1234"

[channel.discord]
enabled = true
token = "1234"
allow_bots = true
notify_channel = "1234"

[provider]
active = "openrouter"

[provider.openrouter]
api_key = "1234"
model = "anthropic/claude-3-haiku"
vision = true

[mounts]
test = "~/Documents/experiment"

There are a few simple changes we can make for Brook compliance.

First, just apply this simple top-level change: tool_call_summaries = "minimal". This complies with the latest codifications on tool messages.

For further compliance, you can consider removing notify_channel, disabling reasoning_summary, or setting allow_bots to false, but at the time of writing, the minimum requirement for your Brook compliance is to add tool_call_summaries.

Additional Guidelines

Brook regulatory guidelines also state:

Limit AI response length to 2 paragraphs as the maximum in most channels, but strongly recommend them to stick to 1-2 sentences for most messages

You are recommended to talk your agent through this and for it to add these terms to its files.
